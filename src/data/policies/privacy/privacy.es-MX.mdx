## II. Finalidades de tratamiento de datos
Se usan los datos recabados mencionados anteriormente para analizar y filtrar
el tráfico web de orígenes fraudulentos en caso de ataques.

La dirección IP es recolectada y anonimizada por Nginx, el servidor proxy
encargado de recibir y controlar el tráfico web de la página web.

A continuación, se presenta la configuración de Nginx usada para garantizar
dichas finalidades:

```conf
map $remote_addr $ip_anonym1 {
    default 0.0.0;
    "~(?P<ip>(\d+)\.(\d+)\.(\d+))\.\d+" $ip;
    "~(?P<ip>[^:]+:[^:]+):" $ip;
}

map $remote_addr $ip_anonym2 {
    default .0;
    "~(?P<ip>(\d+)\.(\d+)\.(\d+))\.\d+" .0;
    "~(?P<ip>[^:]+:[^:]+):" ::;
}

map $ip_anonym1$ip_anonym2 $ip_anonymized {
    default 0.0.0.0;
    "~(?P<ip>.*)" $ip;
}

log_format anonymized '$ip_anonymized - $remote_user [$time_local] "$request" '
    '$status $body_bytes_sent '
    '"$http_user_agent" "$http_x_forwarded_for"';


access_log  /var/log/nginx/access.log anonymized;
```

Dichos datos son almacenados durante un día en un archivo de registro y son
eliminados posteriormente.

Los datos recabados no son usados para finalidades secundarias.
